## Brand Overview

### Mission
Help people find the perfect gifts through AI-powered recommendations that understand what makes each gift special.

### Vision
Become the world's most trusted gift discovery platform by combining technology with genuine thoughtfulness.

### Values
- **Thoughtful:** Every recommendation is carefully considered
- **Personal:** We understand that every gift-giving occasion is unique
- **Trustworthy:** Honest recommendations, transparent affiliate relationships
- **Delightful:** Gift-giving should be joyful, not stressful
- **Inclusive:** Great gifts for everyone, regardless of budget or occasion

## Voice & Tone

### Brand Voice

**Consistent Across All Communications:**

**Conversational**
- Use "you" and "we" to create connection
- Write like you speak (but with better grammar)
- Avoid jargon and overly technical language

**Helpful**
- Lead with what benefits the user
- Provide context and explanations
- Anticipate questions and answer them

**Confident**
- Be direct and clear
- Make recommendations, don't waffle
- Own our expertise in gift recommendations

**Warm**
- Show empathy for gift-giving stress
- Celebrate special occasions
- Express genuine care for user success

### Tone Variations

**By Context:**

| Context | Tone Example |
|---------|-------------|
| **Homepage** | Excited, welcoming: "Find the perfect gift in minutes!" |
| **Search Results** | Confident, helpful: "Here are 10 thoughtful gifts for your friend..." |
| **Error Messages** | Empathetic, solution-focused: "Oops! Let's try that again..." |
| **Success States** | Celebratory, supportive: "Great choice! They'll love it." |
| **Educational Content** | Informative, friendly: "Here's what makes this gift special..." |
| **Legal/Policy** | Clear, respectful: "Here's how we protect your information..." |

### Writing Guidelines

**Do:**
- ‚úÖ Start with the benefit: "Save time finding the perfect gift"
- ‚úÖ Use active voice: "We curate personalized recommendations"
- ‚úÖ Be specific: "87% of users found their perfect gift"
- ‚úÖ Keep it concise: One idea per sentence
- ‚úÖ Use contractions: "We'll help you" not "We will help you"

**Don't:**
- ‚ùå Use corporate speak: "Leverage our synergistic platform"
- ‚ùå Be vague: "Some people like this"
- ‚ùå Overuse exclamation points: "Amazing!!! Perfect!!!"
- ‚ùå Talk about ourselves: "Our award-winning algorithm..."
- ‚ùå Use all caps: "BEST GIFTS EVER"

## Visual Identity

### Primary Brand Color

**Warm Coral #FF6B6B**
- Friendly, approachable, warm
- Evokes gift-wrapped packages and celebration
- Stands out without being aggressive
- Works well with photography

**Usage:**
- Primary CTAs and buttons
- Links and interactive elements
- Brand accents
- Important highlights

### Secondary Brand Color

**Trustworthy Blue #339AF0**
- Reliable, stable, professional
- Complements coral without competing
- Provides visual balance
- Works for data visualization

**Usage:**
- Secondary actions
- Informational elements
- Data displays
- Supporting graphics

## Typography

### Font Family

**Inter** - Sans-serif typeface
- Clean, modern, highly legible
- Excellent at small sizes
- Great OpenType features
- Wide language support

**Installation:**
```html
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800&display=swap" rel="stylesheet">
```

### Type Scale

| Name | Size | Line Height | Usage |
|------|------|-------------|-------|
| Display XL | 48px | 1.0 | Hero headlines |
| Display LG | 36px | 1.1 | Page titles |
| Display MD | 30px | 1.2 | Section headers |
| Display SM | 24px | 1.3 | Large headlines |
| H1 | 36px | 1.2 | Page headings |
| H2 | 30px | 1.3 | Section headings |
| H3 | 24px | 1.4 | Subsection headings |
| H4 | 20px | 1.4 | Card titles |
| H5 | 18px | 1.5 | Small headings |
| H6 | 16px | 1.5 | Overlines |
| Body LG | 18px | 1.6 | Large body text |
| Body | 16px | 1.5 | Default body text |
| Body SM | 14px | 1.4 | Small body text |
| Caption | 12px | 1.3 | Captions, labels |

### Font Weights

- **Light (300):** Rarely used
- **Regular (400):** Body text
- **Medium (500):** Emphasized text
- **Semibold (600):** Subheadings
- **Bold (700):** Headings
- **Extrabold (800):** Display text

### Typography Best Practices

**Do:**
- ‚úÖ Use 16px minimum for body text
- ‚úÖ Maintain 1.5 line height for readability
- ‚úÖ Limit line length to 60-80 characters
- ‚úÖ Use headings hierarchically (H1 ‚Üí H2 ‚Üí H3)
- ‚úÖ Increase letter spacing for uppercase text

**Don't:**
- ‚ùå Use more than 3 font weights per page
- ‚ùå Set body text below 14px
- ‚ùå Use ALL CAPS for long text
- ‚ùå Stack headings without body text between
- ‚ùå Use font sizes that skip hierarchy

## Spacing & Layout

### Grid System

**Base Unit:** 4px
- All spacing is a multiple of 4px
- Creates visual harmony and consistency
- Easier for developers to implement

### Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| 0 | 0px | No spacing |
| 1 | 4px | Tight spacing |
| 2 | 8px | Small spacing |
| 3 | 12px | Default inline spacing |
| 4 | 16px | Default block spacing |
| 6 | 24px | Medium spacing |
| 8 | 32px | Large spacing |
| 12 | 48px | Section spacing |
| 16 | 64px | Page section spacing |
| 24 | 96px | Large page spacing |

### Border Radius

| Token | Value | Usage |
|-------|-------|-------|
| sm | 2px | Small elements |
| DEFAULT | 4px | Buttons, inputs |
| md | 6px | Cards |
| lg | 8px | Large cards |
| xl | 12px | Modal dialogs |
| 2xl | 16px | Hero sections |
| 3xl | 24px | Large images |
| full | 9999px | Pills, avatars |

### Layout Guidelines

**Container Widths:**
- Small: 640px (mobile)
- Medium: 768px (tablet)
- Large: 1024px (desktop)
- XL: 1280px (wide desktop)
- 2XL: 1536px (max width)

**Responsive Breakpoints:**
```
sm:  640px   // Mobile landscape
md:  768px   // Tablet portrait
lg:  1024px  // Tablet landscape / Small desktop
xl:  1280px  // Desktop
2xl: 1536px  // Large desktop
```

## Iconography

### Icon Style

**Lucide Icons** (Preferred)
- Clean, consistent 24√ó24px stroke icons
- 2px stroke width
- Rounded caps and joins
- Available in React, Vue, Svelte

### Icon Sizes

| Size | Dimensions | Usage |
|------|------------|-------|
| xs | 16√ó16px | Inline with text |
| sm | 20√ó20px | Small buttons |
| base | 24√ó24px | Default size |
| lg | 32√ó32px | Large buttons |
| xl | 48√ó48px | Empty states |

### Icon Usage Rules

**Do:**
- ‚úÖ Use consistent stroke width (2px)
- ‚úÖ Align icons optically with text
- ‚úÖ Provide ARIA labels for icon-only buttons
- ‚úÖ Use semantic colors (e.g., green for success)

**Don't:**
- ‚ùå Mix different icon styles
- ‚ùå Use icons smaller than 16px
- ‚ùå Rely on icons alone (add text labels)
- ‚ùå Rotate or skew icons

## Usage Guidelines

### Logo Usage

*Note: Actual logo to be designed in Task 2.0*

**Clear Space:**
- Minimum clear space = height of logo
- No other elements in clear space
- Applies to all logo variations

**Minimum Sizes:**
- Print: 20mm width
- Digital: 120px width
- Favicon: 32√ó32px

### Accessibility

**WCAG 2.1 AA Compliance Required:**
- Text contrast: 4.5:1 minimum
- Large text: 3:1 minimum
- Interactive elements: 3:1 minimum
- Focus indicators: visible and clear
- Touch targets: 44√ó44px minimum

### Don'ts

**Never:**
- ‚ùå Change brand colors
- ‚ùå Use unapproved fonts
- ‚ùå Distort or rotate logo
- ‚ùå Place logo on busy backgrounds
- ‚ùå Use colors outside the palette
- ‚ùå Ignore spacing guidelines
- ‚ùå Skip accessibility requirements
